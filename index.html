VPN / WIREGUARD01

VPN между офисом и удалённым сервером

ЗАДАЧА
Организовать защищённый доступ удалённого сервера к внутренней сети клиента.
ДИАГНОСТИКА
Определили схему адресации, необходимые подсети и направление маршрутизации.
РЕШЕНИЕ
Развернули WireGuard, настроили peers, AllowedIPs, маршруты и форвардинг между сетями.
Стабильный защищённый доступ к инфраструктуре без публикации внутренних сервисов в интернет.
MIKROTIK / NETWORK02

Диагностика и оптимизация сети

ЗАДАЧА

ДИАГНОСТИКА
Проверили интерфейсы, маршрутизацию, NAT, правила Firewall и прохождение трафика.
РЕШЕНИЕ
Скорректировали сетевую конфигурацию и логику маршрутизации.
Стабильная работа сети и предсказуемое прохождение трафика.
VPN / VPS / ROUTING03

Доступ в сеть клиента через VPS

ЗАДАЧА
Связать удалённый VPS с локальной инфраструктурой клиента и обеспечить двусторонний доступ.
ДИАГНОСТИКА
Проверили WireGuard-конфигурацию, IP forwarding, NAT и обратные маршруты.
РЕШЕНИЕ
Настроили wg0 на VPS, маршрутизацию подсетей и корректный форвардинг трафика.
VPS стал полноценной точкой защищённого доступа к нужным ресурсам внутренней сети.
MIKROTIK / LTE04

Разбор падения скорости на LTE-подключении

ЗАДАЧА
После подключения через LTE клиент столкнулся с заметным снижением скорости за MikroTik.
ДИАГНОСТИКА
Проверили конфигурацию маршрутизатора и отделили сетевые настройки от факторов мобильного канала.
РЕШЕНИЕ
Сформировали план проверки LTE-сигнала, агрегации, загрузки базовой станции и фактической пропускной способности канала.
Проблема локализована по уровням: маршрутизатор, LTE-модем и операторский канал проверяются отдельно.
SECURITY / WINDOWS SERVER05

Мониторинг сервера с публичным IP

ЗАДАЧА
Повысить наблюдаемость Windows Server с внешним IP без изменения рабочей сетевой схемы.
ДИАГНОСТИКА
Проверили активные соединения, Firewall, неуспешные входы и состояние критичных сервисов.
